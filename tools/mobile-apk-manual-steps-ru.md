# Алхимик: пошаговая инструкция ручной проверки APK на телефоне

## 1. Что подготовить заранее
1. Установить APK:
`allchemist-release-20260503-0122-mobile-ui.apk`

2. Иметь под рукой три телефона для входа:
- ученик: `+79991110101`
- родитель: `+79991110102`
- преподаватель: `+79991110103`

3. Если нужен код подтверждения:
- его проще смотреть через web:
`http://91.197.99.201:8000/api/v1/web`

4. Проверять лучше по одному сценарию за раз:
- сначала ученик,
- потом родитель,
- потом преподаватель.

## 2. Общая проверка после установки
### 2.1 Установка
1. Открой APK на телефоне.
2. Нажми `Установить`.
3. После установки нажми `Открыть`.

### 2.2 Первый экран
Проверь:
1. Видно название `Алхимик`.
2. Нет старого названия `Synapse`.
3. Фон выглядит научно, а не пусто.
4. Текст читается.
5. Нет вылетов.

Если что-то не так, зафиксируй:
- какой экран,
- что именно не так,
- скриншот.

## 3. Сценарий: Ученик
### 3.1 Онбординг
1. На экране выбора роли нажми `Я ученик/студент`.
2. Проверь, что после выбора не открылся интерфейс родителя или учителя.

### 3.2 Вход по телефону
1. В блоке синхронизации введи телефон:
`+79991110101`
2. Нажми `Получить код`.
3. Получи код через web или через debug-источник.
4. Введи код.
5. Нажми `Подтвердить`.
6. Нажми `Продолжить`, если нужно.

Проверь:
1. Нет ошибки входа.
2. После входа открывается главный экран ученика.

### 3.3 Главный экран ученика
Проверь, что видны:
1. логотип и hero-блок,
2. модули,
3. AI-наставник,
4. вход в live по коду,
5. кнопка подписки,
6. нет смешанных английских строк.

### 3.4 Открыть химию
1. Нажми `Химия`.
2. Проверь главный экран химии.
3. Нажми `Уроки и задачи`.
4. Выбери класс.
5. Выбери учебник.
6. Выбери тему.

Проверь:
1. тема открывается,
2. текст не пустой,
3. кнопка перехода к заданиям работает,
4. интерфейс на русском.

### 3.5 Открыть задачи по химии
1. Нажми `Перейти к заданиям по теме`.
2. Отметь первую задачу как `Правильно` или `Ошибка`.
3. Нажми `Далее`.
4. Дойди до конца мини-сценария.

Проверь:
1. прогресс идет,
2. блок `Разбор ошибок` появляется,
3. приложение не вылетает.

### 3.6 Открыть физику
1. Вернись на главный экран.
2. Нажми `Физика`.
3. Нажми `Уроки`.
4. Нажми `Отчет`.

Проверь:
1. тексты по-русски,
2. нет визуальной каши,
3. нет поломанных кнопок.

### 3.7 Проверка live для ученика
1. На главном экране в блоке live введи код урока, если он уже создан учителем.
2. Введи класс, например `8A`.
3. Нажми `Войти в live`.

Проверь:
1. вход проходит,
2. нет ошибки,
3. если ошибка есть, зафиксируй текст.

## 4. Сценарий: Родитель
### 4.1 Сброс и новый вход
1. Выйди из текущего аккаунта.
2. Снова открой экран выбора роли.
3. Нажми `Я родитель`.

### 4.2 Вход
1. Введи телефон:
`+79991110102`
2. Нажми `Получить код`.
3. Введи код.
4. Нажми `Подтвердить`.

### 4.3 Главный экран родителя
Проверь:
1. видно, что это именно режим родителя,
2. есть кабинет родителя,
3. есть прогресс ребенка,
4. есть план дня,
5. есть рекомендации или риск.

### 4.4 Кабинет родителя
1. Открой `Кабинет`.
2. Найди блок плана дня.
3. Найди прогресс.
4. Найди рекомендации.

Проверь:
1. тексты по-русски,
2. нет служебных английских хвостов,
3. сценарий выглядит как родительский, а не ученический.

### 4.5 Подписка родителя
1. Открой `Тарифы и подписка`.
2. Найди семейный тариф.
3. Проверь описание.

Проверь:
1. название тарифа на русском,
2. описание понятно,
3. нет обещаний, которых пока нет в продукте.

## 5. Сценарий: Преподаватель
### 5.1 Сброс и новый вход
1. Выйди из аккаунта родителя.
2. На выборе роли нажми `Я преподаватель`.

### 5.2 Вход
1. Введи телефон:
`+79991110103`
2. Нажми `Получить код`.
3. Введи код.
4. Нажми `Подтвердить`.

### 5.3 Главный экран преподавателя
Проверь:
1. видны teacher-инструменты,
2. есть доступ к live,
3. есть аналитика,
4. интерфейс не выглядит как ученический.

### 5.4 Teacher live
1. Открой `Кабинет`.
2. Найди блок live.
3. Нажми `Запустить live + QR`.

Проверь:
1. появился `sessionId`,
2. появился `join code`,
3. появился `QR`,
4. нет вылета.

### 5.5 Roster
1. В блоке live найди `Редактор списка класса`.
2. Введи класс `8A`.
3. Введи несколько строк, например:
`student_001`
`student_002`
4. Нажми `Сохранить список`.

Проверь:
1. нет ошибки,
2. кнопки подписаны по-русски,
3. логика понятна.

### 5.6 Обновление live
1. Нажми `Обновить live`.
2. Посмотри блоки:
- участники,
- высокий риск,
- heatmap,
- ошибки.

Проверь:
1. текст читаем,
2. русский UI,
3. нет пустых непонятных блоков.

## 6. Проверка подписки и сохранения прогресса
Это частично уже подтверждено технически, но надо проверить пользовательски.

### 6.1 Базовая проверка
1. Под учеником открой премиум-функцию или экран подписок.
2. Посмотри текущий активный план.
3. Запомни текущий прогресс по одной теме.

### 6.2 После отзыва доступа
Если будет подготовлен отдельный revoke-pass:
1. После отзыва снова открой приложение.
2. Открой тот же модуль.

Проверь:
1. premium-контент закрыт предсказуемо,
2. прогресс не обнулился,
3. нет пустого экрана вместо понятного ограничения.

### 6.3 После возврата доступа
1. После возврата доступа снова открой ту же тему.
2. Проверь прогресс и доступ.

Проверь:
1. прогресс тот же,
2. доступ вернулся,
3. не надо проходить все заново.

## 7. Русский UI: обязательный ручной проход
Пройди и просто визуально проверь:
1. Главная
2. Подписки
3. Кабинет
4. Аналитика
5. Физика
6. Химия
7. Уроки химии
8. Задачи химии
9. Реакции 3D
10. Галерея молекул

На каждом экране ответь:
1. Все ли понятно?
2. Все ли по-русски?
3. Нет ли старого бренда?
4. Нет ли англоязычных product-строк?
5. Нет ли пустого состояния без объяснения?

## 8. Что считать блокером
Сразу считать блокером, если:
1. APK не ставится.
2. Приложение падает.
3. Не работает вход.
4. Не открывается главный сценарий роли.
5. Прогресс исчезает.
6. Live не стартует у учителя.
7. Русский UI явно смешан с англоязычными строками на основных экранах.

## 9. Как фиксировать результат после ручного прохода
Для каждой роли записать:
1. Что прошло.
2. Что не прошло.
3. Где именно проблема.
4. Блокер или неблокер.
5. Нужен ли hotfix.

## 10. Что делать после этой проверки
Если ручной проход успешен:
1. повысить реальный прогресс этапа стабилизации,
2. зафиксировать результаты в append-only журнале,
3. переходить к chemistry redesign.

Если ручной проход неуспешен:
1. сначала закрыть blockers,
2. пересобрать APK,
3. повторить этот же сценарий.
