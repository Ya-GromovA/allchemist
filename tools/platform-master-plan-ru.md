# Алхимик: единый мастер-план продукта (школа + вуз + монетизация)

## 1) Продуктовая цель

Собрать единую образовательную платформу по естественным наукам (сейчас: Химия + Физика, позже: Биология), которая:

- помогает школьнику и студенту учить теорию, решать задачи, выполнять лабораторные сценарии и готовиться к экзаменам;
- дает учителю инструменты выдачи заданий и контроля прогресса группы;
- дает родителю прозрачную аналитику прогресса ребенка;
- работает на мобильных устройствах разной мощности и поддерживает web-режим для школы/аудитории;
- монетизируется через подписку, покупку модулей и B2B-лицензии.

## 2) Роли и ключевые сценарии

### 2.1 Ученик/студент

- выбирает трек (школа/вуз), класс/курс, учебник и автора;
- изучает тему (теория + медиа + интерактив);
- выполняет задачи/тесты/викторины;
- выполняет виртуальную лабораторную работу в 2 режимах реакций;
- получает разбор ошибок и правильные ответы с обоснованием;
- получает экзаменационные вопросы/варианты после изучения блока;
- может отправлять задания/викторины друзьям и при согласии получать их результаты.

### 2.2 Учитель

- создает класс/группу, выдает задания по теме/дате;
- расшаривает задания по коду/ссылке/QR;
- запускает демонстрационный режим урока;
- видит прогресс: кто выполнил, точность, время, типовые ошибки, понимание темы.

### 2.3 Родитель

- видит прогресс по темам, время занятий, результаты;
- видит зоны риска и динамику по неделям;
- получает уведомления о невыполненных заданиях и просадке успеваемости.

## 3) Главные разделы приложения

- **Уроки**
- **Молекулы**
- **Реакции**
- **Задачи**
- **Экзамен**
- **Прогресс**
- **AI-помощник** (доступен из любого экрана)

## 4) Химия: ядро функциональности

### 4.1 Уроки (школа + вуз)

Контент строится по схеме:

- Уровень (школа/вуз) -> Класс/Курс -> Учебник/Автор -> Тема -> Урок/Практика.

В уроке должны быть:

- теория (авторский контент по программе, без копирования защищенных текстов);
- формулы, схемы, иллюстрации, таблицы;
- мини-проверки после блоков;
- связь с задачами и лабораторными сценариями.

### 4.2 Молекулы

- 2D/3D просмотр, масштаб, подписи атомов, связи;
- русские названия, формулы, свойства, классы вещества;
- реакции и совместимость/несовместимость;
- источники информации.

### 4.3 Реакции в двух режимах

1. **Молекулярный режим**
   - пользователь соединяет молекулы;
   - видно разрыв/образование связей, распад, присоединение, растворение;
   - есть текстовое объяснение происходящего и условий реакции.

2. **Пробирки/колбочки**
   - пользователь смешивает вещества, нагревает/охлаждает, меняет порядок добавления;
   - визуализация результата (осадок, газ, изменение цвета, кипение, взрыв и т.д.);
   - мультимодальные эффекты: звук (шипение/всплеск/взрыв), вибрация (события повышенной интенсивности);
   - предупреждения по ТБ и рискам.

### 4.4 Лабораторные задания от учителя

- учитель выдает обязательный сценарий лабораторной;
- ученик проходит шаги и фиксирует результат;
- доступен sandbox для дополнительных экспериментов (добавить реагент, изменить условия) с последствием в симуляции;
- учитель видит факт выполнения, корректность шагов, итог и ошибки.

## 5) Контент для студентов: полный охват дисциплин

Помимо базовой химии включаются:

- общая химия;
- неорганическая химия;
- органическая химия;
- аналитическая химия;
- физическая химия;
- коллоидная химия;
- биохимия;
- химическая технология;
- лабораторный практикум и безопасность.

Для каждой дисциплины: теория, задачи, практики, лабораторные сценарии, контроль и экзаменационные блоки.

## 6) AI-помощник из любого окна

### 6.1 Доступность

- плавающая AI-кнопка/панель в любом разделе;
- текстовый ввод + голосовой ввод (диктовка);
- контекстный вопрос по текущему экрану/теме/задаче.

### 6.2 Режимы AI

- Offline AI: локальный поиск и объяснение на базе загруженного контента;
- Online AI: расширенные ответы при наличии интернета;
- автоматический fallback online -> offline.

### 6.3 Учебные режимы

- объяснить тему;
- разобрать решение пошагово;
- проверить ответ;
- сгенерировать дополнительные вопросы/варианты в рамках изученного блока.

## 7) Экзамены, тесты, викторины и социализация

- генерация экзаменационных вопросов после завершения теории;
- генерация вариантов по шаблонам + банку проверенных заданий;
- показ ошибок и правильных ответов с обоснованием;
- отправка задач/викторин друзьям;
- получение результата отправителем только по согласию решающего.

## 8) Прогресс и аналитика

### 8.1 Ученик/студент

- прогресс по темам/модулям;
- mastery-score по навыкам;
- история ошибок и рекомендации.

### 8.2 Учитель

- дашборд класса: выполнение, средний балл, время, сложные темы;
- отчеты по домашним/лабораторным/контрольным;
- ранжирование тем по уровню усвоения.

### 8.3 Родитель

- индивидуальная карточка ребенка;
- недельный отчет и динамика;
- уведомления о рисках и просроченных задачах.

## 9) Лицензии и легальный контент

### 9.1 Базовый принцип

- не копировать защищенные учебники дословно без лицензии;
- использовать авторский контент по учебной программе, сохраняя корректность и учебные цели;
- обязательно хранить источники и лицензии контента.

### 9.2 Политика источников

- фактическая информация: открытые лицензии, официальные справочники, научные базы;
- у каждого блока: `source`, `license`, `verified_by`, `updated_at`;
- аудит контента перед публикацией.

## 10) Архитектура данных и производительности

### 10.1 Почему не вшивать все в APK

- большой вес и медленные обновления;
- ухудшение работы на слабых устройствах.

### 10.2 Рекомендованный подход

- thin-client + скачиваемые контент-модули;
- first-run загрузка обязательного core-пака;
- затем выборочная загрузка предметов/классов/дисциплин;
- фоновое обновление модулей и версионирование паков;
- offline-first хранение: SQLite + индексы + FTS;
- тяжелые ассеты (3D/аудио) через CDN и ленивую подгрузку.

### 10.3 Первый запуск

- экран прогресса загрузки (`Подготовка данных`);
- выбор набора модулей (минимум / школа / вуз / все);
- опция `только Wi-Fi`;
- запуск в базовом режиме до завершения полной догрузки.

## 11) Монетизация

### 11.1 Модель дохода

- Freemium (бесплатное ядро + ограниченный контент);
- подписки (1/3/12 месяцев);
- one-time покупка модулей;
- B2B-лицензии для школ/вузов;
- AI-квоты и докупка пакетов online-запросов.

### 11.2 Что платное

- полный доступ к модулям и дисциплинам;
- расширенные лабораторные и реакции;
- экзаменационный генератор;
- AI Pro (увеличенный лимит online-запросов);
- учительский кабинет и расширенная аналитика.

## 12) Платежная архитектура (РФ)

Поддерживаем платежи через:

- **Robokassa**
- **T-Bank (Тинькофф) Эквайринг**
- **YooKassa**

### 12.1 Принцип интеграции

- единый `Payment Gateway Abstraction Layer` на backend;
- провайдер выбирается по каналу/региону/доступности;
- общий жизненный цикл: `init -> pay -> webhook -> verify -> grant entitlement`;
- idempotency для вебхуков и транзакций;
- журнал аудита платежных событий.

### 12.2 Подписки и покупки

- mobile IAP (сторы) + web-платежи через провайдеры;
- one-time покупки модулей;
- рекуррентные подписки (где доступно);
- семейные и школьные тарифы.

### 12.3 Безопасность платежей

- подпись/проверка вебхуков;
- хранение только необходимых платежных данных;
- secrets в защищенном хранилище, ротация ключей;
- антифрод-правила и лимиты попыток.

## 13) Web-режим и масштабирование

- единый backend для mobile/web;
- web-кабинет учителя и web-решение задач по ссылке;
- демонстрация материалов на уроках через браузер;
- синхронизация прогресса между устройствами;
- подготовка к добавлению модуля `Биология` на той же архитектуре.

## 14) Дорожная карта реализации

### Этап A (MVP Foundation)

- роли, auth, entitlements, прогресс;
- core-модули, first-run загрузка, каталог курсов;
- базовая монетизация и платежная абстракция.

### Этап B (Chemistry Full)

- химия 7-11 + вузовые дисциплины (приоритетные);
- молекулы 2D/3D;
- реакции: молекулярный + колбочки;
- лабораторные сценарии и разбор ошибок.

### Этап C (Teacher/Parent)

- выдача заданий, классы, шаринг;
- аналитика для учителя;
- кабинет родителя.

### Этап D (Exam + AI)

- генератор экзаменов;
- AI online/offline во всех окнах + голос;
- адаптивные рекомендации по слабым темам.

### Этап E (Web + Scale)

- web-клиент ученика и учителя;
- B2B-лицензирование школ;
- подготовка подключения модуля Биология.

## 15) Критерии успеха

- высокая завершенность уроков и заданий;
- рост удержания и платящей аудитории;
- снижение времени учителя на проверку;
- прозрачная динамика прогресса для родителей;
- стабильная работа на слабых устройствах и при плохой сети.

## 15.1 Обязательные governance-блоки

В MVP обязательно включаются и проходят DoD-проверку:

- юрчасть (оферта, персональные данные несовершеннолетних, родительские согласия);
- античит и правила честного прохождения;
- эксплуатация (SLO/SLA, мониторинг, алерты, бэкапы, восстановление);
- модерация user-generated контента (шаринг задач/результатов);
- web-roadmap по ролям (ученик/учитель/админ);
- unit-economics и A/B paywall.

Детализация по реализации, этапам и DoD вынесена в документ: `docs/mvp-backlog-ru.md`.

## 18) Обязательные продуктовые блоки качества и доверия

### 18.1 Доказательство учебной эффективности

- измеряем не только engagement, но и рост учебных результатов;
- запускаем A/B по learning outcomes (контрольные метрики до/после обучения);
- фиксируем improvement по темам, средний рост точности и скорость решения задач.

### 18.2 Контент-операционка

- определены роли: автор, научный редактор, методист, QA-контента;
- SLA на контент-ошибки и сроки обновления тем;
- обязательная научная валидация перед публикацией и журнал изменений контента.

### 18.3 LTV-удержание

- персональные планы обучения;
- ежедневные цели и цепочки прогресса;
- механики возврата без агрессивного paywall.

### 18.4 Онбординг по ролям

- отдельный first-time сценарий для ученика/студента;
- отдельный first-time сценарий для учителя;
- отдельный first-time сценарий для родителя.

### 18.5 Fail-safe для слабых условий

- graceful degradation графики и анимаций;
- fallback режимы при слабой сети/низкой мощности;
- приоритет стабильной учебной функции над визуальными эффектами.

### 18.6 Trust-слой

- в каждом AI-ответе: источник, дата обновления, уровень уверенности;
- прозрачный маркер: ответ из локальной базы или из online-источников;
- возможность перейти к исходному материалу.

## 19) Дифференцирующие фичи (включены в roadmap)

- Персональный ИИ-репетитор с памятью прогресса и планом "до контрольной за N дней".
- Режим "Сдай экзамен": таймер, реалистичный формат, стресс-режим, разбор.
- AI-объяснение в 3 уровнях: простой, студенческий, экзаменационный.
- Лаборатория с последствиями: sandbox-эксперименты + ТБ и объяснение последствий.
- Режим "Учитель в классе за 1 минуту": QR-выдача и live-статус.
- Режим "Родитель без перегруза": один экран ежедневных действий.
- Карьерный мост: применение темы в реальных профессиях и индустриях.
- Анти-выгорание UX: микро-сессии, адаптивная сложность, маленькие победы.

## 20) Быстрые win-фичи для ближайших релизов

- автоматический план подготовки к контрольной/сессии;
- teacher live dashboard во время урока;
- источники/достоверность в каждом AI-ответе;
- умные карточки ошибок с интервальным повторением;
- реферальная механика "кинь другу задание".

## 16) UI/UX: анимации, переходы, легкость интерфейса

### 16.1 Визуальный принцип

- интерфейс должен выглядеть современно, но оставаться легким и быстрым;
- ключевой фокус: читаемость контента и быстрый доступ к действию;
- анимации усиливают понимание процессов, а не перегружают экран.

### 16.2 Обязательные анимации

- плавные переходы между экранами и карточками;
- анимированное раскрытие теории, подсказок, разборов ошибок;
- мягкие переходы в молекулярном 2D/3D и в режиме колбочек;
- интерактивные эффекты при правильном/неправильном ответе;
- плавные skeleton/loading экраны при догрузке контента.

### 16.3 Производительность анимаций

- режимы качества: `Lite`, `Standard`, `Enhanced`;
- на слабых устройствах: сниженный FPS и упрощенные эффекты;
- отключение тяжелых частиц/света при низком ресурсе GPU;
- все критические анимации не должны блокировать ввод пользователя.

### 16.4 Реакции: мультимодальность

- визуал + текстовое объяснение происходящего;
- звуковое сопровождение событий реакции (шипение, кипение, взрыв и т.п.);
- вибрация на сильных событиях (с учетом настроек устройства и согласия пользователя);
- опция отключения звука/вибрации отдельно для классов/аудиторий.

## 17) Операционный регламент: сборка APK и установка

Этот блок нужен для быстрого восстановления процесса, если чат/сессия прервется.

### 17.1 Сборка и публикация APK на сервере

```bash
ssh -i ~/.ssh/wishhunter_server root@91.197.99.201
cd /root/synapse/mobile/android
./gradlew --no-daemon assembleRelease
cp -f /root/synapse/mobile/android/app/build/outputs/apk/release/app-release.apk /root/synapse/content_packs/synapse-layer-a-debug-20260220.apk
chmod 644 /root/synapse/content_packs/synapse-layer-a-debug-20260220.apk
sha256sum /root/synapse/content_packs/synapse-layer-a-debug-20260220.apk
```

### 17.2 Скачивание и установка на Windows

```powershell
Remove-Item ".\synapse-layer-a-debug.apk" -Force -ErrorAction SilentlyContinue
$ts = [DateTimeOffset]::UtcNow.ToUnixTimeSeconds()
Invoke-WebRequest "http://91.197.99.201:8000/api/v1/content/downloads/apk/layer-a-debug?t=$ts" -OutFile ".\synapse-layer-a-debug.apk"
Get-FileHash ".\synapse-layer-a-debug.apk" -Algorithm SHA256
adb uninstall com.synapse.mobile
adb install ".\synapse-layer-a-debug.apk"
```

### 17.3 Быстрая проверка после установки

- приложение запускается без Metro/без ошибки `Unable to load script`;
- открываются разделы `Уроки`, `Молекулы`, `Реакции`, `Задачи`;
- проверяется свежесть APK по SHA256;
- проверяется базовый пользовательский сценарий (открыть тему -> решить задачу -> увидеть прогресс).
