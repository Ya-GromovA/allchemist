# Алхимик: подробное ТЗ на redesign модуля химии

## 1. Контекст
Текущий модуль химии технически существует, но продуктово не устраивает.

Главная проблема:
- экранов уже много,
- но глубина теории и связность учебного маршрута слабые,
- нет полноценного режима по классам,
- нет полноценного режима по учебной линии,
- не хватает визуализации,
- нет полноценного экзаменационного контура.

## 2. Цель redesign
Сделать химию главным сильным модулем платформы.

Итоговый модуль должен давать:
1. Понятную теорию по уровню класса.
2. Дополнительный режим по учебной линии.
3. Переход от теории к практике без разрыва.
4. Визуализацию процессов.
5. Проверочные работы и демо-экзамены.
6. Удобный интерфейс для ученика, родителя и учителя.

## 3. Основные проблемы текущего состояния
### 3.1 Контент
- теория слишком тонкая,
- нет полноценного покрытия по всем классам,
- контент местами похож на заглушку,
- структура по учебникам пока полуформальная.

### 3.2 Навигация
- сейчас есть выбор класс -> учебник -> тема,
- но маршрут слишком утилитарный,
- не хватает ощущения цельного учебного курса.

### 3.3 Практика
- задачи есть,
- но связка теория -> тренировка -> контроль -> разбор пока недостаточно сильная.

### 3.4 Визуализация
- molecules/reactions есть,
- но они пока воспринимаются как отдельные экраны,
- а не как естественная часть учебного объяснения.

## 4. Принцип построения теории
Нужно два режима теории.

### 4.1 Режим 1. Базовая теория по классу
Назначение:
- безопасный авторский контент,
- объяснение на уровне конкретного класса,
- единая каноническая версия без зависимости от учебника.

Требования:
- для 7, 8, 9, 10, 11 классов,
- по темам программы,
- с нарастающей сложностью,
- с простым и полным уровнем объяснения.

### 4.2 Режим 2. Теория по учебной линии
Назначение:
- соответствовать реальному учебнику пользователя,
- не нарушая авторское право.

Требования:
- не копировать защищенный текст дословно,
- использовать оригинальный пересказ,
- следовать структуре нужной линии:
  - тема,
  - порядок,
  - терминология,
  - типовые примеры,
  - логика подачи.

### 4.3 Правовая модель
Нужно разделить два уровня:
1. `Обычный режим`
- оригинальный контент по мотивам программы и структуры.

2. `Лицензируемый режим`
- только при наличии договоренностей с правообладателями,
- для изображений,
- иллюстраций,
- фирменных таблиц,
- близких к учебнику формулировок.

## 5. Экранная архитектура модуля химии
### 5.1 Главный экран химии
Должен содержать:
- название раздела,
- класс,
- выбранный режим теории,
- выбранную учебную линию,
- текущий прогресс,
- главный следующий шаг.

Блоки:
1. Продолжить тему
2. Теория
3. Практика
4. Проверочная работа
5. Демо ЕГЭ / ОГЭ / МЦКО
6. Реакции и визуализация
7. AI-наставник по текущей теме

### 5.2 Экран выбора маршрута
Пользователь выбирает:
1. Класс
2. Режим
- Базовая теория
- По учебной линии
3. Если выбран второй режим:
- учебная линия,
- автор,
- учебник,
- часть / глава

### 5.3 Экран темы
Состав экрана:
1. Заголовок темы
2. Короткая цель
3. Что должен понять ученик
4. Вкладки:
- Теория
- Иллюстрации
- Анимации
- Задачи
- Проверка
- Ошибки и разбор

### 5.4 Экран теории
Внутри темы теория должна делиться на блоки:
1. Понятие
2. Объяснение простым языком
3. Формулы и обозначения
4. Частые ошибки
5. Где это встречается в заданиях
6. Мини-проверка после блока

### 5.5 Экран визуализации
Виды визуализации:
1. Схема
2. Иллюстрация
3. Анимация процесса
4. 2D-модель
5. 3D-модель
6. Пошаговая реакция

### 5.6 Экран практики
Режимы:
1. Тренировочные задачи
2. 5 быстрых задач
3. Углубленные задачи
4. Задачи на типовые ошибки

### 5.7 Экран проверки знаний
1. Мини-проверка по теме
2. Контрольная по разделу
3. Итоговая работа по классу

### 5.8 Экран демо-экзаменов
Отдельные режимы:
1. Демо ОГЭ
2. Демо ЕГЭ
3. Демо МЦКО

Для каждого режима:
- таймер,
- структура вариантов,
- проверка,
- разбор ошибок,
- рекомендации по слабым местам.

### 5.9 Экран викторины
1. Один против одного
2. Классная викторина
3. Викторина от учителя
4. Быстрый блиц по теме

## 6. Контентная модель
Для каждой темы химии нужны:
1. `theory_short`
2. `theory_full`
3. `theory_by_grade`
4. `theory_by_bookline`
5. `key_terms`
6. `formula_block`
7. `common_mistakes`
8. `exam_links`
9. `visual_assets`
10. `animation_steps`
11. `practice_sets`
12. `mini_check`
13. `teacher_notes`
14. `parent_notes`

## 7. Визуальный стиль химии
### 7.1 Общая идея
Химия должна выглядеть как сильный научный модуль, а не как набор карточек.

### 7.2 Визуальные опоры
- молекулярные сетки,
- цветовые кодировки веществ,
- лабораторные слои,
- аккуратные схемы,
- графика процессов,
- карточки без перегруза.

### 7.3 Правила интерфейса
1. Один главный шаг на экран.
2. Теория не должна быть сплошным полотном текста.
3. После каждого смыслового блока должна быть микро-проверка или переход к визуализации.
4. Практика должна быть связана с объяснением, а не отдельно от него.

## 8. Сценарии по ролям в модуле химии
### 8.1 Ученик
Видит:
- текущую тему,
- теорию,
- визуализацию,
- задачи,
- демо-экзамены,
- викторины.

Главный маршрут:
теория -> пример -> тренировка -> мини-проверка -> разбор ошибок -> следующий шаг.

### 8.2 Родитель
Видит:
- какой раздел проходит ребенок,
- что уже понял,
- где слабые места,
- что сделать сегодня.

### 8.3 Учитель
Видит:
- темы для показа классу,
- live-демонстрации реакций,
- готовые проверочные,
- викторины,
- аналитику ошибок по теме.

## 9. Платные функции в химии
### Базовый доступ
- часть теории,
- базовые задачи,
- базовые реакции,
- базовый прогресс.

### Расширенный доступ
- полная теория,
- теория по учебной линии,
- больше визуализации,
- демо-экзамены,
- расширенные задачи,
- дополнительные AI-разборы.

### Важное правило
Если пользователь купил chemistry-пакет:
- состав пакета должен быть четко описан,
- после обновления функционала он не должен получить меньше, чем обещали на момент покупки.

## 10. Что нужно реализовать по данным и системе
1. Расширить chemistry content pack.
2. Разделить:
- базовую теорию,
- учебные линии,
- экзаменационные задания,
- визуальные assets.
3. Добавить версионирование chemistry content.
4. Добавить legal metadata для контента по учебным линиям.
5. Добавить teacher/parent notes в структуру темы.

## 11. Что нужно проверить после реализации
### Контент
- тема не пустая,
- теория реально полезная,
- класс соответствует уровню,
- учебная линия похожа на нужный учебный маршрут.

### UX
- ученик не теряется,
- родитель не попадает в ученический шум,
- учитель быстро доходит до показа и проверки.

### Визуализация
- иллюстрации помогают понять тему,
- анимации не декоративные, а обучающие,
- реакции встроены в объяснение.

### Монетизация
- paywall не ломает маршрут,
- premium объяснен честно,
- прогресс не теряется при окончании доступа.

## 12. Определение готовности
Модуль химии можно считать действительно переработанным только если:
1. Есть полный маршрут по нескольким классам.
2. Есть два режима теории.
3. Есть визуальный слой.
4. Есть практика и проверка.
5. Есть демо-экзамены.
6. Есть role-based ценность для ученика, родителя и учителя.
7. Нет юридически сомнительного копирования учебников.

## 13. Следующий практический шаг
После утверждения этого ТЗ следующая реализация должна идти не по хаотичным правкам, а по очереди:
1. Информационная архитектура химии.
2. Контентная схема темы.
3. Экран выбора режима теории.
4. Новый экран темы.
5. Новый теоретический слой.
6. Новый visual layer.
7. Практика и проверочные.
8. Демо-экзамены.
9. Викторины.
