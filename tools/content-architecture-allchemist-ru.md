# Алхимик: контентная архитектура по предметам, уровням и ролям

## 1. Главный принцип дальнейшей разработки
Дальше двигаемся не от подписок и не от точечных экранов, а от правильной учебной архитектуры.

Правильный порядок такой:
1. Зафиксировать единую контентную модель.
2. Определить учебные маршруты по предметам и уровням.
3. Сформировать UX-модель экранов и переходов.
4. Наполнить контент по предметам.
5. Подключить визуализацию, тактильность, аудио и лабораторные эффекты.
6. После этого уже настраивать роли, права, тарифы и ограничения.

Причина:
- если сначала делать доступы и подписки, а потом менять структуру контента, придется ломать уже выданные права и пользовательские сценарии;
- если сначала построить учебную архитектуру, то роли и подписки лягут на нее аккуратно.

## 2. Новый рабочий порядок разработки

### Этап 1. Контентная архитектура
Сначала описываем общую структуру контента для всех предметов:
1. базовая теория;
2. теория по учебной линии;
3. визуальные материалы;
4. задачи;
5. проверочные работы;
6. демо-экзамены;
7. викторины;
8. заметки родителю;
9. заметки учителю;
10. научные источники и provenance.

### Этап 2. Наполнение предметов
Приоритет наполнения:
1. химия;
2. физика;
3. биология.

### Этап 3. Логика приложения
После наполнения формируем:
1. маршруты переходов;
2. роли;
3. права доступа;
4. подписки;
5. school access;
6. family access.

### Этап 4. Монетизация
Только после того, как понятно, что именно реально есть в продукте, вводим:
1. student packages;
2. advanced chemistry packages;
3. school licenses;
4. parent analytics packages.

## 3. Контентная модель для каждого предмета
Для каждой темы любого предмета должны существовать следующие слои.

### 3.1 Базовая теория
Это главный безопасный и обязательный слой.

Содержит:
1. краткое объяснение;
2. полное объяснение;
3. ключевые термины;
4. формулы / определения / обозначения;
5. типичные ошибки;
6. мини-проверку после блока.

### 3.2 Теория по учебной линии
Нужна для соответствия реальному учебнику пользователя.

Содержит:
1. название линии;
2. автора;
3. тему/параграф;
4. оригинальный пересказ по логике учебника;
5. порядок тем, соответствующий линии;
6. legal metadata.

Важно:
- не копируем учебник дословно;
- используем оригинальный пересказ на основе официальных источников и структуры линии.

### 3.3 Визуальный слой
Содержит:
1. иллюстрации;
2. схемы;
3. инфографику;
4. таблицы;
5. 2D/3D-визуализацию;
6. пошаговые анимации.

### 3.4 Практика
Содержит:
1. простые задачи;
2. стандартные задачи;
3. углубленные задачи;
4. задачи на типичные ошибки;
5. объяснение решений.

### 3.5 Проверка знаний
Содержит:
1. мини-проверку;
2. проверочную по теме;
3. проверочную по разделу;
4. итоговую работу.

### 3.6 Экзаменационный слой
Содержит:
1. Демо ОГЭ;
2. Демо ЕГЭ;
3. Демо МЦКО;
4. таймер;
5. итог;
6. разбор ошибок;
7. маршрут возврата к слабым темам.

### 3.7 Игровой слой
Содержит:
1. личные викторины;
2. классные викторины;
3. live-викторины учителя;
4. соревновательные блицы.

## 4. Архитектура наполнения по предметам

### 4.1 Химия
Химия делится на два больших контура:

#### А. Школьная химия
1. базовая теория по классам;
2. теория по учебной линии;
3. школьные задачи;
4. проверочные;
5. ОГЭ/ЕГЭ/МЦКО;
6. реакции и лабораторные сценарии.

#### Б. Расширенные направления химии
Отдельные content-модули:
1. аналитическая химия;
2. физколлоидная химия;
3. биохимия;
4. органическая химия углубленно;
5. неорганическая химия углубленно;
6. промышленная/технологическая химия;
7. лабораторное оборудование и процессы.

Пример маршрута:
- если пользователь открывает дефлегматор, его нужно вести в раздел физколлоидной/технологической химии, а не в школьную тему без контекста.

### 4.2 Физика
1. школьная физика по классам;
2. теория по учебной линии;
3. визуальные модели процессов;
4. эксперименты и лабораторные сценарии;
5. экзаменационный слой;
6. углубленные ветки:
   - механика,
   - электричество,
   - оптика,
   - термодинамика,
   - атомная физика.

### 4.3 Биология
1. школьная биология по классам;
2. теория по учебной линии;
3. схемы строения и процессов;
4. проверочные;
5. экзаменационный слой;
6. расширенные ветки:
   - анатомия,
   - генетика,
   - биохимия,
   - экология,
   - микробиология.

## 5. Источники и достоверность
Контент нельзя строить на выдуманных объяснениях.

Нужно опираться на:
1. официальные школьные программы;
2. официальные учебные линии;
3. вузовские учебные материалы;
4. научные справочники;
5. официальные методические материалы;
6. академические источники.

Для каждой темы желательно хранить metadata:
1. источник;
2. тип источника;
3. дата редакции;
4. проверено ли редактором;
5. соответствует ли школьному уровню / вузовскому уровню.

## 6. Дизайн и удобство для пользователя

### 6.1 Общие принципы дизайна
Интерфейс должен быть:
1. спокойным для глаз;
2. читаемым;
3. логичным;
4. не перегруженным;
5. тактильно приятным;
6. предсказуемым по навигации.

### 6.2 Цветовая гамма
Рекомендуемая база:
1. темный глубокий фон без визуального шума;
2. мягкие акцентные цвета;
3. отдельные акцентные коды для предметов:
   - химия: янтарно-бирюзовый / лабораторный;
   - физика: сине-электрический;
   - биология: зеленый натуральный.

Важно:
- фон не должен содержать навязчивый текстовый шум;
- декоративные слои должны быть почти незаметными;
- приоритет — читабельность и структура, а не спецэффекты ради спецэффектов.

### 6.3 Расположение кнопок и опций
На каждом основном экране:
1. один главный CTA;
2. сверху короткий контекст — где я и что делаю;
3. ниже основной рабочий блок;
4. вторичные действия только после главного.

### 6.4 Личный кабинет
Личный кабинет должен быть одним из самых удобных экранов.

Он должен содержать:
1. профиль;
2. роль;
3. статус подписки;
4. состояние прогресса;
5. настройки темы;
6. настройки стиля заголовков;
7. переключатели визуализации / звука / вибрации;
8. сетевой статус;
9. историю устройств и синхронизации.

## 7. Тактильность, звук и визуальные эффекты

### 7.1 Где использовать
В лабораторных работах и реакциях:
1. вспышка;
2. вибрация;
3. звук реакции;
4. эффект газа / осадка / взрыва / кипения / искр.

### 7.2 Как использовать правильно
Эффекты должны:
1. усиливать обучение,
2. быть привязаны к событию,
3. не быть навязчивыми,
4. отключаться пользователем.

### 7.3 Обязательные настройки в кабинете
1. Вибрация эффектов: вкл/выкл
2. Звуковые эффекты: вкл/выкл
3. Интенсивность эффектов: мягко / стандарт / ярко
4. Анимации: сокращенные / стандартные / расширенные

## 8. Роли и доступы: пока только архитектурно
Ты предложила сначала не ограничивать роли доступами, а сначала наполнить приложение. Это разумно.

Поэтому сейчас роли не режем по подпискам, а только закладываем будущую модель:

### 8.1 Ученик
- школьный маршрут,
- проверочные,
- экзамены,
- базовые и расширенные модули.

### 8.2 Студент
- отдельная роль,
- расширенные вузовские ветки,
- углубленные модули.

### 8.3 Родитель
- базовый просмотр прогресса,
- результаты работ,
- ошибки и пояснения,
- рекомендации.

Рекомендация:
- базовый родительский доступ действительно лучше делать бесплатным.

### 8.4 Учитель
- классы,
- проверочные,
- викторины,
- назначение работ,
- live-сценарии,
- аналитика ошибок.

### 8.5 Школа
- отдельная модель доступа,
- лицензия на школу / классы / места,
- централизованное управление.

## 9. Правильный следующий порядок разработки

### Шаг 1
Зафиксировать контентную схему для всех предметов.

### Шаг 2
Начать системное наполнение:
1. химия школьная;
2. химия по учебным линиям;
3. расширенные ветки химии;
4. физика;
5. биология.

### Шаг 3
Параллельно доводить UX-маршруты:
1. тема;
2. теория;
3. визуализация;
4. практика;
5. проверка;
6. разбор ошибок.

### Шаг 4
После наполнения и логики:
1. роли;
2. доступы;
3. подписки;
4. school licensing.

## 10. Что считать готовностью контентной архитектуры
Архитектура готова, если:
1. она едина для всех предметов;
2. поддерживает школьный и расширенный контур;
3. не нарушает авторское право;
4. позволяет строить красивый, тактильный, визуально сильный UX;
5. не ломает дальнейшую модель подписок и ролей.
