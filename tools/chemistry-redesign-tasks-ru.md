# Алхимик: chemistry redesign как набор задач разработки

## 1. Принцип разбиения
Каждая задача описана в формате:
- экран,
- данные,
- UI,
- DoD.

Это нужно, чтобы redesign химии шёл итерациями и можно было честно считать прогресс по подтвержденным результатам.

## 2. Эпик A. Новая архитектура химии

### Задача A1. Экран входа в модуль химии
Экран:
- новый главный экран химии.

Данные:
- текущий класс,
- выбранный режим теории,
- выбранная учебная линия,
- текущий прогресс по химии,
- доступные разделы.

UI:
- hero-блок,
- прогресс,
- главный CTA `Продолжить тему`,
- карточки:
  - Теория,
  - Практика,
  - Проверка,
  - Демо-экзамены,
  - Реакции и визуализация,
  - AI-наставник.

DoD:
- пользователь понимает, где он находится,
- есть один главный следующий шаг,
- экран читается на мобильном без перегруза,
- нет смешения старого chemistry shell и нового маршрута.

### Задача A2. Экран выбора режима теории
Экран:
- выбор режима `Базовая теория` / `По учебной линии`.

Данные:
- режим,
- класс,
- доступные учебные линии,
- доступные авторы,
- доступность по тарифу.

UI:
- переключатель двух режимов,
- объяснение разницы,
- селектор класса,
- селектор учебной линии.

DoD:
- пользователь понимает разницу между режимами,
- логика выбора не запутывает,
- paywall не ломает выбор.

## 3. Эпик B. Теоретический слой

### Задача B1. Базовая теория по классу
Экран:
- экран темы, вкладка `Теория`.

Данные:
- theory_short,
- theory_full,
- ключевые термины,
- формулы,
- типовые ошибки,
- мини-проверка.

UI:
- карточки по смысловым блокам,
- режим `Кратко` / `Подробно`,
- блок `Важно запомнить`,
- блок `Частые ошибки`.

DoD:
- по теме есть не заглушка, а полноценное объяснение,
- контент соответствует уровню класса,
- после прочтения можно перейти к практике без потери контекста.

### Задача B2. Теория по учебной линии
Экран:
- экран темы, вкладка `По учебной линии`.

Данные:
- учебная линия,
- автор,
- глава,
- тема,
- оригинальный пересказ,
- mapping к структуре учебника.

UI:
- явная маркировка учебной линии,
- подпись, что это адаптированное объяснение,
- переход между соседними темами линии.

DoD:
- не используется запрещенное дословное копирование,
- пользователь видит, что маршрут соответствует нужному учебнику,
- контент редакционно целостный.

### Задача B3. Parent/Teacher notes для темы
Экран:
- дополнительные секции внутри темы.

Данные:
- подсказка родителю,
- подсказка учителю,
- цель урока,
- на что обратить внимание.

UI:
- компактные раскрывающиеся карточки.

DoD:
- родитель и учитель получают отдельную ценность,
- основной ученический экран не перегружен.

## 4. Эпик C. Визуализация

### Задача C1. Экран иллюстраций и схем
Экран:
- вкладка `Иллюстрации`.

Данные:
- изображения,
- подписи,
- схемы,
- простые комментарии.

UI:
- слайдер карточек,
- zoom-friendly просмотр,
- подписи под изображениями.

DoD:
- изображения реально помогают понять тему,
- не являются декоративным шумом.

### Задача C2. Экран анимаций
Экран:
- вкладка `Анимации`.

Данные:
- animation_steps,
- captions,
- связанные термины,
- учебная цель анимации.

UI:
- пошаговая анимация,
- кнопки `Назад`, `Далее`, `Повторить`,
- текстовое объяснение шага.

DoD:
- анимация объясняет, а не просто движется,
- пользователь понимает, что именно происходит на каждом шаге.

### Задача C3. Встраивание молекул и реакций в обучение
Экран:
- теория и реакции больше не отдельные миры.

Данные:
- связка тема -> molecules -> reactions -> visual steps.

UI:
- из теории можно сразу открыть релевантную реакцию,
- из реакции можно вернуться в теорию и практику.

DoD:
- реакции встроены в учебный маршрут,
- нет ощущения разрозненных экранов.

## 5. Эпик D. Практика и проверка

### Задача D1. Новый экран задач по теме
Экран:
- практика по теме.

Данные:
- задачи по теме,
- уровень сложности,
- теги ошибок,
- объяснения решений.

UI:
- фильтры сложности,
- блокировка хаотичного skip,
- понятный статус прогресса.

DoD:
- задача решается в связке с темой,
- после ошибки пользователь получает направление, что повторить.

### Задача D2. Мини-проверка
Экран:
- быстрый контроль по теме.

Данные:
- небольшой набор вопросов,
- scoring,
- слабые места.

UI:
- таймер по желанию,
- результат,
- рекомендации.

DoD:
- после проверки видно, что усвоено,
- система может предложить следующий шаг.

### Задача D3. Разбор ошибок
Экран:
- отдельный блок после практики/мини-проверки.

Данные:
- wrong answers,
- mistake taxonomy,
- привязка к теории.

UI:
- список ошибок,
- `почему ошибка`,
- `что повторить`,
- `попробовать еще раз`.

DoD:
- пользователь понимает не только что ошибка, но и почему.

## 6. Эпик E. Проверочные и экзамены

### Задача E1. Проверочная работа по теме
Экран:
- отдельный режим `Проверочная`.

Данные:
- набор заданий,
- время,
- результаты,
- разбор.

UI:
- старт,
- прохождение,
- итог.

DoD:
- проверочная реально завершима,
- результат сохраняется,
- есть разбор ошибок.

### Задача E2. Демо ОГЭ / ЕГЭ / МЦКО по химии
Экран:
- экран выбора формата экзамена.

Данные:
- демо-варианты,
- структура работы,
- лимит времени,
- результат,
- слабые темы.

UI:
- карточки форматов,
- экран прохождения,
- экран итога.

DoD:
- пользователь может пройти демо как тренажер,
- итог связан с реальными темами химии,
- можно вернуться к теории по слабым местам.

## 7. Эпик F. Викторины

### Задача F1. Личная викторина по теме
Экран:
- быстрый игровой режим.

Данные:
- вопросы,
- время,
- счет,
- рейтинг попытки.

UI:
- яркий темп,
- короткие раунды,
- итоговая карточка.

DoD:
- режим короткий и не утомляет,
- может использоваться как повторение темы.

### Задача F2. Учительская викторина по классу
Экран:
- teacher launch screen + student participation.

Данные:
- класс,
- набор вопросов,
- live results.

UI:
- запуск,
- join,
- scoreboard,
- победители.

DoD:
- учитель реально может провести быструю проверку на уроке,
- ученики понимают сценарий без лишних объяснений.

## 8. Эпик G. Данные и контент

### Задача G1. Новая chemistry content schema
Экран:
- не экранная задача, а data layer.

Данные:
- theory blocks,
- bookline mapping,
- visuals,
- animation steps,
- exam links,
- practice sets,
- parent notes,
- teacher notes.

UI:
- нет прямого UI, но структура должна поддерживать все новые экраны.

DoD:
- схема позволяет хранить все нужные типы контента без костылей.

### Задача G2. Наполнение по классам
Экран:
- влияет на все chemistry screens.

Данные:
- минимум базовое покрытие 7-11 классов.

UI:
- темы доступны в маршруте класса.

DoD:
- по каждому классу есть не пустой маршрут тем.

### Задача G3. Наполнение по учебным линиям
Экран:
- режим по учебной линии.

Данные:
- автор,
- линия,
- тема,
- оригинальный пересказ,
- legal flags.

UI:
- показывается как отдельный режим.

DoD:
- минимум 1-2 полноценно проработанные линии как proof of concept.

## 9. Эпик H. Монетизация и доступ

### Задача H1. Paywall для chemistry
Экран:
- chemistry gates.

Данные:
- что бесплатно,
- что premium,
- что входит в school access,
- что входит в family.

UI:
- мягкий paywall,
- объяснение ценности,
- переход к тарифам.

DoD:
- пользователь понимает, зачем покупать,
- учебный маршрут не ломается полностью.

### Задача H2. Заморозка и восстановление прогресса
Экран:
- chemistry progress state after entitlement changes.

Данные:
- progress storage,
- frozen access state,
- restore behavior.

UI:
- понятное сообщение при окончании доступа,
- понятное сообщение после продления.

DoD:
- прогресс не теряется,
- после продления пользователь продолжает с того же места.

## 11. Как считать реальный прогресс chemistry redesign
Прогресс считать только по задачам, которые:
1. реализованы,
2. визуально проверены,
3. прошли ручной сценарий,
4. подтверждены контентно, а не просто пустыми данными.
