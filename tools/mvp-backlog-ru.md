# Алхимик: структурированный MVP backlog (v1)

## 0) Что еще не обсудили до конца

- юридическая модель и оферта: лицензия на контент, пользовательское соглашение, политика конфиденциальности;
- персональные данные несовершеннолетних: согласия родителя, сроки хранения, удаление аккаунта;
- античит и честность прохождения (особенно для школьных/вузовских контрольных);
- SLO/SLA и эксплуатация: мониторинг, алерты, бэкапы, disaster recovery;
- модерация пользовательского контента (шаринг заданий/результатов);
- roadmap по web-версии в разрезе ролей (ученик/учитель/админ школы);
- unit-economics: CAC/LTV, лимиты online-AI по тарифам, paywall A/B тесты.

## 1) Принципы MVP

- запуск с фокусом на **Химию** (школа + базовый вузовый контур), Физика остается активной, Биология — следующий этап;
- mobile-first, offline-first, модульная загрузка контента;
- один backend и единая модель данных для будущего web-клиента;
- монетизация включена с первого релиза (freemium + подписка + модули).

## 2) Эпики MVP и приоритет

### E1. Платформенное ядро (P0)

- роли: ученик/студент, учитель, родитель;
- auth + привязка родителя к ученику;
- entitlement-модель (подписки, купленные модули, AI-квоты);
- first-run загрузка core-пака.

### E2. Контент и уроки (P0)

- иерархия: уровень -> класс/курс -> учебник/автор -> тема;
- каталог тем (школа 7–11 + стартовый вуз);
- авторский теоретический контент + источники/лицензии;
- экран урока с прогрессом изучения.

### E3. Задачи и проверка (P0)

- банк задач по темам и сложности;
- попытка решения + проверка;
- разбор ошибок и правильный ответ с обоснованием;
- мини-квиз по завершению темы.

### E4. Реакции в 2 режимах (P0)

- молекулярный режим (базовые сценарии реакций + описание);
- колбочки/пробирки (смешивание, нагрев, базовые эффекты);
- звук и вибрация с настройками;
- базовые лабораторные сценарии.

### E5. Молекулы (P1)

- 2D/3D просмотр, масштаб, подписи атомов;
- формула, название, свойства, совместимость;
- поиск и фильтрация;
- связь молекулы с уроками и задачами.

### E6. AI в любом окне (P1)

- плавающий AI-виджет;
- текстовый ввод + диктовка;
- offline AI (локальный поиск/объяснение) + online AI fallback;
- контекстные подсказки по текущему экрану.

### E7. Учитель и родитель (P1)

- выдача задания классу по ссылке/коду;
- просмотр выполнения и прогресса класса;
- родительский отчет по ребенку (тема/время/ошибки).

### E8. Монетизация и платежи (P0)

- тарифы: free / subscription / module purchase;
- платежный шлюз-абстракция;
- провайдеры: Robokassa, T-Bank, YooKassa;
- webhook-подтверждение и выдача прав доступа.

### E9. Learning Effectiveness (P0)

- метрики learning outcomes (до/после по темам);
- A/B эксперименты по учебным сценариям;
- дашборд прироста успеваемости.

### E10. Контент-операционка (P0)

- workflow: автор -> научред -> методист -> контент QA -> публикация;
- SLA на исправление ошибок контента;
- журнал изменений контента и статусы валидации.

### E11. Retention и персональные планы (P1)

- ежедневные цели и цепочки;
- персональный план подготовки (до контрольной/сессии);
- умные карточки ошибок (повтор по интервалам).

### E12. Trust и надежность UX (P0)

- trust-слой в AI-ответах (источник, уверенность, дата);
- role-based onboarding (ученик/учитель/родитель);
- fail-safe профили (сеть/девайс) и graceful degradation.

### E13. Differentiator features (P1)

- персональный AI-репетитор с памятью прогресса;
- режим "Сдай экзамен";
- AI-объяснение в 3 уровнях;
- лаборатория с последствиями;
- teacher live dashboard;
- parent quick insight;
- карьерный мост;
- анти-выгорание UX.

## 3) Спринтовый план (8 спринтов)

## Sprint 1 (P0 Foundation)

- схема ролей и entitlements;
- first-run загрузка core-модуля;
- базовый каталог курсов/тем;
- telemetry: events, crashes, perf;
- role-based onboarding skeleton;
- fail-safe profile detection (Lite by default on weak devices).

**DoD:** новый пользователь проходит онбординг и видит доступные темы по роли.

## Sprint 2 (Уроки + контент)

- экран выбора класс/курс + учебник/автор;
- экран темы и урока;
- контент-пак формат v2 (source/license fields);
- офлайн-кэш уроков;
- content-operations workflow + SLA fields.

**DoD:** урок открывается офлайн после загрузки модуля.

## Sprint 3 (Задачи + разбор)

- движок попыток и оценивания;
- экран результатов, ошибок и обоснований;
- мини-квизы по теме;
- шаринг задания другу (MVP link);
- trust-слой для AI-ответов (source/confidence/updated_at).

**DoD:** после решения пользователь видит ошибки и корректные решения.

## Sprint 4 (Реакции v1)

- молекулярный режим: базовые сценарии;
- колбочки/пробирки: базовые сценарии;
- эффекты звук/вибрация + настройки;
- протокол безопасности/предупреждений;
- лаборатория с последствиями (sandbox v1).

**DoD:** 10+ стабильных реакционных сценариев в обоих режимах.

## Sprint 5 (Молекулы + UX)

- карточка молекулы (2D/3D), связи, подписи;
- улучшение производительности на слабых девайсах;
- поиск с быстрым сбросом;
- анимации/переходы в стандарте Lite/Standard;
- anti-burnout UX: микро-сессии и маленькие победы.

**DoD:** 60 FPS на средних устройствах, Lite режим на слабых.

## Sprint 6 (AI everywhere)

- AI-виджет на всех экранах;
- голосовой ввод (диктовка);
- offline/online маршрутизация;
- лимиты online-запросов по тарифу;
- AI-объяснение в 3 уровнях;
- персональный AI-репетитор (memory + слабые темы).

**DoD:** вопрос можно задать из любого экрана голосом или текстом.

## Sprint 7 (Учитель/родитель)

- выдача заданий классу, дедлайны;
- дашборд учителя по теме/классу;
- родительская карточка прогресса;
- согласие на передачу результатов между учениками;
- teacher live dashboard на уроке;
- parent one-screen daily plan.

**DoD:** учитель видит, кто сделал задание и с каким результатом.

## Sprint 8 (Монетизация + Payments + Release)

- paywall и тарифы;
- интеграция Robokassa / T-Bank / YooKassa;
- проверка вебхуков и выдача entitlements;
- релиз-кандидат + регресс + release checklist;
- learning outcomes A/B + paywall A/B + unit-economics dashboard.

**DoD:** покупка/подписка корректно открывает платный контент и AI-квоты.

## 4) Бэклог по данным и производительности

- контент-паки по предметам/классам/дисциплинам;
- дедупликация ассетов;
- lazy loading 3D и медиа;
- целевые размеры: базовый APK <= 90MB, остальное модулями;
- cold start budget <= 3 сек на средних девайсах;
- офлайн-доступ к последним загруженным темам.

## 5) MVP метрики

- D1/D7 retention;
- lesson completion rate;
- test accuracy uplift;
- teacher assignment completion rate;
- paid conversion (trial -> paid);
- online AI cost per active user.

## 6) Риски MVP и смягчение

- перегрузка функциональностью -> жесткий приоритет P0/P1;
- рост стоимости online-AI -> квоты и fallback на offline;
- тяжелая графика -> Lite режим и деградация эффектов;
- юридические риски контента -> source/license pipeline + review.

## 7) Юрчасть и персональные данные несовершеннолетних (обязательно в MVP)

### 7.1 Документы и артефакты (P0)

- публичная оферта (платные услуги и цифровой контент);
- пользовательское соглашение;
- политика конфиденциальности;
- политика контента и авторских прав;
- согласие на обработку персональных данных;
- отдельное согласие родителя/законного представителя для несовершеннолетних.

### 7.2 Возраст и согласия

- возрастной gate при регистрации;
- для несовершеннолетнего ограниченный функционал до подтверждения родителя;
- подтверждение родителя по инвайт-коду + подтвержденный канал связи;
- фиксирование версии согласий, даты, IP/device fingerprint в журнале аудита.

### 7.3 Данные и хранение

- data minimization: собираем только необходимое;
- retention policy по типам данных;
- право удаления аккаунта и данных (self-service + SLA обработки запроса);
- экспорт данных пользователя по запросу.

### 7.4 DoD

- все юридические документы доступны из onboarding и профиля;
- без подтверждения согласий нельзя перейти к персонализированным функциям;
- админ видит журнал версий согласий и историю подтверждений.

## 8) Античит и правила честного прохождения

### 8.1 Политика честного режима (P0)

- режимы: `Тренировка` (подсказки доступны) и `Контроль/Экзамен` (подсказки ограничены);
- прозрачная маркировка попытки (самостоятельно/с подсказками/с AI);
- запрет на публикацию ответов контрольных в общий доступ.

### 8.2 Технические меры (P1)

- серверная валидация событий прохождения (тайминг, шаги, аномалии);
- защита от повторной отправки результатов (idempotency tokens);
- device/session signals для выявления массового abuse;
- ограничение частоты генерации экзаменационных вариантов.

### 8.3 Учительский контроль

- флаги подозрительных попыток в отчете учителя;
- раздельная статистика по честным/подсказочным попыткам;
- настройка уровня строгости для контрольной.

### 8.4 DoD

- контрольная попытка имеет статус доверия;
- учитель видит подозрительные сессии в дашборде;
- повторная отправка одной попытки не искажает результаты.

## 9) Эксплуатация: SLO/SLA, мониторинг, алерты, бэкапы

### 9.1 Целевые SLO (MVP)

- API availability: `99.5%`/месяц;
- p95 latency для ключевых endpoint: `<= 700ms`;
- crash-free sessions mobile: `>= 99.0%`;
- успешная доставка контент-пака: `>= 98%` попыток.

### 9.2 SLA для инцидентов

- P1 (критично, платежи/логин/контент недоступны): реакция `<= 15 мин`;
- P2 (деградация ключевых функций): реакция `<= 1 час`;
- P3 (некритичные дефекты): реакция `<= 1 рабочий день`.

### 9.3 Observability стек

- centralized logs (backend + mobile telemetry);
- metrics (RPS, latency, error-rate, payment success-rate);
- traces по платежам и выдаче entitlements;
- crash/error monitoring для mobile/web.

### 9.4 Алерты и runbooks

- алерты по 5xx, росту latency, падению конверсии платежа, ошибкам webhooks;
- runbook для каждого P1/P2 сценария;
- on-call график и postmortem шаблон.

### 9.5 Бэкапы и DR

- ежедневный full backup + hourly incremental для БД;
- шифрование backup;
- тест восстановления не реже 1 раза в месяц;
- RPO/RTO для MVP: `RPO <= 1 час`, `RTO <= 4 часа`.

## 10) Модерация UGC (шаринг задач/результатов)

### 10.1 Политика контента

- запрещены оскорбления, токсичность, персональные данные третьих лиц, спам;
- запрещен шаринг контрольных ответов в открытом виде;
- правила видимости результата: только по согласию решающего.

### 10.2 Поток модерации

- pre-filter (автофильтры по словам/паттернам/ссылкам);
- report flow (жалоба пользователя);
- queue для ручной модерации;
- санкции: скрытие контента, ограничение шаринга, временная блокировка.

### 10.3 DoD

- кнопка жалобы доступна в shared-контенте;
- все жалобы имеют статус и SLA обработки;
- user receives moderation decision notification.

## 11) Web-roadmap по ролям (ученик/учитель/админ)

### 11.1 Фаза Web-1 (ученик, MVP)

- вход/профиль;
- открытие тем и задач по ссылке учителя;
- выполнение и отправка результата;
- просмотр личного прогресса.

### 11.2 Фаза Web-2 (учитель)

- создание классов/групп;
- выдача заданий/лабораторных;
- просмотр прогресса и ошибок по группе;
- демонстрационный режим урока.

### 11.3 Фаза Web-3 (админ школы)

- управление лицензиями и пользователями;
- отчеты по классам/параллелям;
- управление интеграциями и политиками доступа;
- аудит действий и выгрузка отчетности.

### 11.4 Зависимости

- единая RBAC-модель и entitlement-service;
- единый API контракт для mobile/web;
- SSO/инвайты для школьных доменов (следующий этап).

## 12) Unit-economics и A/B paywall

### 12.1 Обязательные метрики

- CAC по каналам;
- activation rate (первый урок + первая задача);
- trial-to-paid conversion;
- ARPPU / ARPU;
- churn (30/60 дней);
- LTV/CAC ratio;
- online-AI cost per paying user.

### 12.2 Пейволл эксперименты (MVP)

- A/B #1: момент показа paywall (после 1-й темы vs после 1-го теста);
- A/B #2: пакетирование (подписка-only vs подписка+модули);
- A/B #3: оффер AI (лимит в базовом vs trial AI Pro);
- A/B #4: месячный vs квартальный фокус в CTA.

### 12.3 Guardrails

- не ухудшать учебный UX ради краткосрочной конверсии;
- ограничение стоимости online-AI на активного пользователя;
- mandatory quality threshold для платного контента.

### 12.4 DoD

- дашборд unit-economics доступен команде продукта;
- A/B система фиксирует эксперименты и статистическую значимость;
- решения по тарифам принимаются на основе метрик, а не интуиции.
