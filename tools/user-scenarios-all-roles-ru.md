# Пользовательские сценарии Алхимик (все роли)

## 1) Ученик

1. Входит в приложение (без SMS, через школьный аккаунт/пароль или код школы).
2. На главной видит учебные модули и AI-наставника.
3. Получает push-приглашение в live, открывает уведомление, код подставляется в форму.
4. Подключается к live, решает задачи по физике/химии.
5. Результаты сразу попадают в аналитику и кабинет.
6. В кабинете меняет тему/профиль производительности/режим сети.

## 2) Учитель

1. Входит в роль преподавателя.
2. Запускает live-сессию, получает join code + QR.
3. Заполняет roster класса (classroom + student IDs).
4. Нажимает "Отправить push классу", ученики получают приглашение.
5. Следит за live-аналитикой: heatmap, risk, mistakes, roster coverage.
6. По очереди intervention queue открывает нужные уроки/задачи.

## 3) Родитель

1. Входит как родитель, видит дашборд ребенка.
2. Открывает one-screen daily plan.
3. Смотрит риск-уроки, слабые темы и learning effectiveness.
4. Получает рекомендованные шаги на неделю (повтор + практика + мини-рефлексия).
5. Отслеживает динамику и вовлеченность ребенка по дням.

## 4) Администратор/оператор

1. Контролирует health/SLO/alerts.
2. Публикует новый APK в release endpoint.
3. Проверяет backup и периодически делает restore-drill.
4. Ведет аудит изменений ролей/прав/контента.

## 5) Платформенный сценарий desktop

1. Пользователь устанавливает Алхимик Desktop (Windows/macOS/Linux).
2. Запускает с иконки рабочего стола.
3. Приложение автоматически проверяет обновления.
4. Учитель ведет live прямо с desktop-клиента.
