# Алхимик: device-level checklist мобильного релиза

## 1. Назначение
Этот документ нужен для ручной проверки APK на реальном Android-устройстве.

Проверяем не только то, что экран открывается, но и:
- что сценарий доходит до конца,
- что роль видит именно свой интерфейс,
- что русский UI не смешан с английским,
- что подписки и прогресс ведут себя предсказуемо.

## 2. Текущий статус этапа
- Этап: Стабилизация мобильного релиза
- Реальный прогресс этапа: около 48%
- Что уже подтверждено технически:
  - login / auth / profile / modules для student, parent, teacher,
  - teacher live -> student join,
  - сохранность progress marker при revoke/grant доступа,
  - сборка release APK,
  - несколько проходов по русскому UI.
- Что еще не подтверждено вручную на устройстве:
  - целостный сценарий по каждой роли,
  - UI после установки APK,
  - конец подписки и восстановление прогресса через пользовательский путь,
  - отсутствие смешанных строк и слабых пустых состояний.

## 4. Общий smoke после установки APK
### 4.1 Установка и первый запуск
- [ ] APK устанавливается без ошибок.
- [ ] Приложение открывается без crash.
- [ ] Splash и главный стартовый экран отображаются корректно.
- [ ] Есть логотип Алхимика.
- [ ] Главный фон выглядит как научный, а не случайный абстрактный.
- [ ] Нет старого бренда Synapse.

### 4.2 Базовая визуальная проверка
- [ ] В главном меню нет английских строк в русском интерфейсе.
- [ ] Кнопки читаемы на маленьком экране.
- [ ] Нет обрезанных текстов.
- [ ] Нет наложения карточек друг на друга.
- [ ] Нет белых экранов при переходах.

### 4.3 Сеть и офлайн
- [ ] При старте онлайн приложение загружает актуальные модули.
- [ ] При отключении сети приложение не падает.
- [ ] В офлайне хотя бы базовый контент доступен.
- [ ] Возврат в онлайн не ломает сессию.

## 5. Сценарий роли: Ученик
### 5.1 Онбординг и вход
- [ ] Выбор роли Ученик понятен.
- [ ] После выбора не открывается teacher/parent интерфейс.
- [ ] Вход по телефону работает, если используется.

### 5.2 Главный экран ученика
- [ ] Видны учебные модули.
- [ ] Есть понятный следующий шаг.
- [ ] Есть AI-наставник.
- [ ] Есть вход в live по коду.

### 5.3 Учебный сценарий
- [ ] Открывается Химия.
- [ ] Открывается Физика.
- [ ] Открывается Биология preview.
- [ ] Открывается AI-наставник.
- [ ] Можно открыть урок.
- [ ] Можно открыть задачи.
- [ ] Можно завершить хотя бы одну задачу.

### 5.4 Прогресс
- [ ] После решения задачи прогресс обновляется.
- [ ] После перезапуска приложения прогресс не исчезает.
- [ ] Прогресс виден в кабинете/отчете.

### 5.5 Подписка
- [ ] Открывается экран подписок.
- [ ] Названия тарифов на русском.
- [ ] Видно, какой тариф активен.
- [ ] Попытка оплаты не ломает сценарий.

## 6. Сценарий роли: Родитель
### 6.1 Онбординг и вход
- [ ] Выбор роли Родитель понятен.
- [ ] Интерфейс отличается от ученического.

### 6.2 Главный экран родителя
- [ ] Есть родительский кабинет.
- [ ] Есть блок прогресса ребенка.
- [ ] Есть план дня.
- [ ] Есть зоны риска или рекомендации.

### 6.3 Содержимое кабинета
- [ ] Тексты на русском.
- [ ] Нет служебных англоязычных названий.
- [ ] Видно, что это именно режим родителя.

### 6.4 Подписка
- [ ] Видно семейный тариф.
- [ ] Пояснение тарифа понятно.
- [ ] Нет ложного обещания недоступного функционала.

## 7. Сценарий роли: Учитель
### 7.1 Онбординг и вход
- [ ] Выбор роли Преподаватель понятен.
- [ ] Интерфейс отличается от student/parent.

### 7.2 Главный экран учителя
- [ ] Видны учительские quick actions.
- [ ] Есть доступ к live / аналитике.
- [ ] Нет ученического домашнего фокуса как главного сценария.

### 7.3 Live-сценарий
- [ ] Учитель может запустить live.
- [ ] Появляется join code.
- [ ] Появляется QR.
- [ ] Можно обновить live-состояние.
- [ ] Отображается список/аналитика участников.

### 7.4 Roster и classroom flow
- [ ] Можно ввести класс.
- [ ] Можно ввести список учеников.
- [ ] Кнопки roster/live подписаны по-русски.

## 8. Сценарий подписки и окончания доступа
### 8.1 Что проверить
- [ ] При активной подписке premium-контент открыт.
- [ ] После отзыва тарифа premium-контент блокируется предсказуемо.
- [ ] После окончания доступа приложение не удаляет прогресс.
- [ ] После возврата доступа прогресс снова доступен.

### 8.2 Что не должно происходить
- [ ] Не должен сбрасываться локальный прогресс.
- [ ] Не должен сбрасываться device sync snapshot.
- [ ] Не должна ломаться роль пользователя.
- [ ] Не должен появляться пустой кабинет вместо понятного paywall/notice.

- [ ] пустые состояния сформулированы по-русски.

## 11. Критерии закрытия этапа стабилизации релиза
Этап нельзя считать завершенным, пока не выполнено всё ниже:
- [ ] полный ручной проход student,
- [ ] полный ручной проход parent,
- [ ] полный ручной проход teacher,
- [ ] ручная проверка подписки/отзыва/возврата доступа,
- [ ] русский UI-проход по основным экранам,
- [ ] установка и запуск свежего APK без regressions,

## 12. Формат фиксации результата проверки
Для каждой сессии ручной проверки сохранить:
- дата и время,
- устройство,
- версия APK,
- какая роль проверялась,
- что прошло,
- что не прошло,
- блокер / неблокер,
- скриншоты проблем,
